# 5 封爛 Email — STRIKE 改造標準答案

> **用途**：講師對答案用，不發給學員
> **對照**：`student_download/02_資訊戰/五封爛Email原稿.md`

## E1 — 回覆客戶延遲交貨

### 爛 Prompt

```
幫我寫一封道歉信給客戶，說貨會遲到。
```

### STRIKE 標準答案

```
【S — 情境】
我是某消費品公司的業務部經理。
客戶是合作三年的大中華區經銷商（王經理）。
這批訂單對客戶的 Q1 鋪貨計畫很重要。

【T — 任務】
撰寫一封客戶道歉 Email，通知交貨延遲。

【R — 角色】
你是擅長客戶關係管理的資深業務，能在道歉的同時維持客戶信任。

【I — 輸入】
- 訂單編號：PO-2026-0312
- 原訂交期：3/10，實際預計：3/17，延遲 7 天
- 原因：供應商原料短缺（非我方品質問題）
- 已安排空運趕工，額外成本由我方吸收
- 寄件人職稱：業務部經理

【K — 驗收指標】
- 語氣：專業但有誠意，不卑不亢
- 結構：開頭致歉 → 說明原因 → 補救方案 → 後續承諾
- 長度：200-300 字
- 格式：Email 格式，含主旨行
- 禁止：不要推卸責任給供應商、不要過度承諾

【E — 範例】
主旨行格式：「交期調整通知：PO-2026-0312 預計延至 3/17 交付」
```

### 評分重點

| 要素 | 必須出現 | 常見遺漏 |
|------|---------|---------|
| S | 客戶關係背景（合作多久、這批貨的重要性） | 只寫「客戶」，沒交代關係深淺 |
| T | 明確寫「道歉 Email」 | 寫「回覆客戶」太模糊 |
| I | 延遲天數、原因、補救措施三項缺一不可 | 最常漏「補救措施」 |
| K | 結構要求（致歉→原因→補救→承諾） | 沒指定結構，AI 自由發揮 |

## E2 — 請求跨部門協助

### 爛 Prompt

```
幫我寫一封信請設計部幫忙。
```

### STRIKE 標準答案

```
【S — 情境】
我是產品部的 PM，目前負責 Q2 新品上市專案。
專案已通過部門主管核准，預計 5 月上線。
需要設計部支援產品頁面和包裝視覺，但設計部不歸我管，
過去沒有直接合作過。

【T — 任務】
撰寫一封跨部門協作請求 Email，爭取設計部的資源支援。

【R — 角色】
你是擅長跨部門溝通的資深 PM，能讓對方感到被尊重而非被指派。

【I — 輸入】
- 需要的設計支援：產品頁面 UI（3 頁）+ 包裝視覺（1 款）
- 期望交付時間：4/15 前交初稿
- 我方能提供：完整的產品 brief、文案定稿、品牌素材包
- 收件人：設計部主管 + CC 我的主管
- 彈性空間：時程可協商，但不晚於 4/25

【K — 驗收指標】
- 語氣：尊重、合作邀請感（不是命令）
- 結構：說明專案背景 → 具體需求（量化）→ 我方能配合什麼 → 期望時程（有彈性）→ 邀請討論
- 長度：200-300 字
- 禁止：不要用「越快越好」、不要用命令句

【E — 範例】
結尾格式：「方便的話，這週找 30 分鐘聊一下需求細節？我可以配合你們的時間。」
```

### 評分重點

| 要素 | 必須出現 | 常見遺漏 |
|------|---------|---------|
| S | 說明專案背景 + 為什麼需要他們 | 直接說「幫忙做設計」沒脈絡 |
| I | 具體需求量化（幾頁、幾款） + 我方能提供什麼 | 只說「一些設計」 |
| K | 語氣要求「邀請感」+ 禁止命令句 | 沒限制語氣，AI 可能寫得像指派 |
| E | 給結尾範例，校準合作邀請的語氣 | 缺 E，AI 結尾可能太生硬 |

## E3 — 拒絕不合理加班

### 爛 Prompt

```
幫我寫一封信拒絕加班。
```

### STRIKE 標準答案

```
【S — 情境】
我是行銷部的資深企劃，主管週五下午五點臨時要求我
週末加班完成一份 Q3 市場趨勢報告。
這份報告的截止日原本是下週三，不是緊急任務。
我和主管的關係正常，想維持良好的工作關係。

【T — 任務】
撰寫一封拒絕週末加班但維持良好關係的回覆 Email。

【R — 角色】
你是擅長向上溝通的資深職場人，
能在拒絕的同時讓主管感到被尊重。

【I — 輸入】
- 任務：Q3 市場趨勢報告
- 原截止日：下週三
- 主管期望：週日晚上交
- 我的狀況：週末已有家庭安排，無法配合
- 我的替代方案：週一早上優先處理，週二下班前交

【K — 驗收指標】
- 語氣：尊重、先認同主管的出發點、再提替代方案
- 結構：感謝交辦 → 說明困難（不說「不想」）→ 替代方案（具體時間）→ 承諾完成品質
- 長度：150-200 字
- 禁止：不說「我不想加班」、不說「這不是我的問題」、不說「這又不急」

【E — 範例】
參考語氣：「謝謝主管提前讓我知道這份報告的重要性。
這個週末我有既定的家庭安排比較難調整，
但我可以週一一早優先處理，週二下班前交出完整版本，
您看這樣的時程可以嗎？」
```

### 評分重點

| 要素 | 必須出現 | 常見遺漏 |
|------|---------|---------|
| R | 指定「擅長向上溝通」的角色 | 沒指定角色，AI 可能寫得太直接 |
| K | 禁止清單（不說不想、不推責） | 沒設禁區，AI 可能寫出得罪人的話 |
| K | 結構要求先認同再拒絕 | 只說「拒絕」，沒指定先認同 |
| E | 給語氣範例，校準分寸感 | 缺 E，AI 拿捏不準職場分寸 |

## E4 — 催主管簽核拖了兩週的案子

### 爛 Prompt

```
幫我寫一封催主管簽核的信。
```

### STRIKE 標準答案

```
【S — 情境】
我是產品部的 PM，一份 Q3 行銷預算案（金額 150 萬）
已經過部門會議通過，兩週前送簽給主管，至今未核。
下游影響：設計部無法啟動視覺製作（已延遲 5 天）、
媒體採購的早鳥價將於下週五截止（錯過多花 20 萬）。

【T — 任務】
撰寫一封內部催辦 Email，推動主管盡快簽核。

【R — 角色】
你是擅長向上管理的資深 PM，
能在催辦的同時展現專業度和配合度。

【I — 輸入】
- 簽核文件：Q3 行銷預算案
- 金額：150 萬
- 送簽日期：2/27，已等待 14 天
- 下游影響 1：設計部視覺製作延遲 5 天
- 下游影響 2：媒體早鳥價下週五截止，逾期多花 20 萬
- 文件狀態：已過部門會議決議

【K — 驗收指標】
- 語氣：尊重但有急迫感，不卑不亢
- 結構：點出文件狀態 → 說明下游影響（量化）→ 請求具體簽核時程 → 表達配合意願
- 長度：150-200 字
- 禁止：不要有抱怨語氣、不要暗示主管拖延
- 主旨行格式：「懇請簽核：Q3 行銷預算案 — 影響設計啟動及媒體採購時程」

【E — 範例】
結尾格式：「如果有需要補充的資料或需要我當面說明，
隨時可以配合您的時間。」
```

### 評分重點

| 要素 | 必須出現 | 常見遺漏 |
|------|---------|---------|
| S | 量化影響（延遲幾天、多花多少錢） | 只寫「影響下游」太模糊 |
| I | 送簽日期 + 等待天數 | 寫「一段時間」沒有說服力 |
| K | 禁止抱怨 + 要求請求具體時程 | 沒設禁區，AI 可能寫出暗諷語氣 |
| K | 主旨行包含影響範圍 | 主旨只寫「請簽核」沒急迫感 |

## E5 — 通知團隊專案時程變更

### 爛 Prompt

```
幫我寫一封通知團隊時程變更的信。
```

### STRIKE 標準答案

```
【S — 情境】
我是專案的 PM，客戶臨時追加了 2 個功能需求，
經評估後交付日需要從 3/20 延到 4/5（延後 16 天）。
團隊有 5 人，各自負責不同模組，時程變更會影響每個人的工作排程。
已和客戶確認新時程，需要通知團隊調整計畫。

【T — 任務】
撰寫一封專案時程變更通知 Email，發給全體專案成員。

【R — 角色】
你是有條理的專案經理，能把複雜的時程變動講清楚，
讓每個人知道自己該做什麼調整。

【I — 輸入】
- 原交付日：3/20 → 新交付日：4/5（延後 16 天）
- 變更原因：客戶追加「多語系支援」和「報表匯出」兩個功能
- 成員與影響：
  - 前端 Amy：多語系 UI 調整，新截止 3/28
  - 後端 Ben：報表 API 開發，新截止 3/25
  - 測試 Cindy：測試案例新增，新截止 4/2
  - 設計 Dan：多語系切版，新截止 3/22
  - 文件 Ella：使用手冊更新，新截止 4/3

【K — 驗收指標】
- 結構：變更摘要（一句話）→ 變更原因 → 各人影響（用表格）→ 下一步行動 → 邀請回報問題
- 格式：含表格（成員 / 負責項目 / 原截止日 / 新截止日）
- 語氣：正式但不僵硬，有團隊感
- 長度：300-400 字
- 禁止：不要歸咎客戶、不要用「之後再說」

【E — 範例】
表格格式：
| 成員 | 負責項目 | 原截止日 | 新截止日 |
|------|---------|---------|---------|
| Amy | 多語系 UI | 3/15 | 3/28 |
| Ben | 報表 API | 3/12 | 3/25 |

結尾格式：「如果新時程有衝突，請在週五前讓我知道，
我們一起想辦法調整。」
```

### 評分重點

| 要素 | 必須出現 | 常見遺漏 |
|------|---------|---------|
| I | 每位成員的具體影響 + 新截止日 | 只說「時程延後」沒人知道自己要幹嘛 |
| K | 指定表格格式 | 沒要求表格，AI 用純文字很難掃讀 |
| E | 給表格範例 + 結尾範例 | 缺 E，AI 格式每次不同 |
| K | 禁止「之後再說」 | 不設禁區，AI 可能模仿原本的爛信語氣 |

## 綜合評分速查

五封信各自測驗的 STRIKE 能力：

| Email | 主要測驗要素 | 次要測驗要素 | 學員最常踩的坑 |
|-------|------------|------------|--------------|
| E1 延遲交貨 | I + K | S | I 只寫「貨會遲到」沒給數據 |
| E2 跨部門協助 | S + K | I | 語氣像命令，沒用 K 限制 |
| E3 拒絕加班 | R + K | E | 沒指定角色，AI 寫得太直白 |
| E4 催簽核 | S + I | K | 沒量化影響，催辦沒力道 |
| E5 時程變更 | I + E | K | 沒給表格範例，AI 產出散亂 |
