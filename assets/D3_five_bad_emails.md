# D3 — 5 封爛 Email 原稿
> **格式**：Google Doc ｜ **用途**：資訊戰（第二單元）實作素材

---

## E1 — 回覆客戶延遲交貨

**情境**：客戶下了一批訂單，交期要延遲 7 天，你要寫信通知。

```
王經理您好，

不好意思，我們的貨可能會晚一點到，
造成您的不便我們很抱歉。

我們會盡快處理。

謝謝
```

**為什麼「爛」**：沒說延遲多久、沒講原因、沒給補救方案、語氣模糊。
**STRIKE 改造重點**：I（具體數據：延遲幾天、原因、補救措施）+ K（結構：致歉→原因→補救→承諾）

## E2 — 請求跨部門協助

**情境**：你的專案需要設計部門支援，但他們不歸你管。

```
Hi 設計部，

我們這邊有個專案需要你們幫忙做一些設計，
可以幫我們處理一下嗎？

時間的話越快越好。

謝謝！
```

**為什麼「爛」**：沒說為什麼需要、沒說要設計什麼、沒給具體時程、語氣像在命令別人。
**STRIKE 改造重點**：S（專案背景、為什麼需要他們）+ K（期望回應方式、具體時程、你能提供什麼配合）

## E3 — 拒絕不合理加班

**情境**：主管週五下午五點丟一個非緊急任務，要你週末加班完成。

```
主管您好，

我這週末沒辦法加班，
不好意思。

謝謝。
```

**為什麼「爛」**：太直白，沒有認同主管的出發點、沒有提替代方案、容易被認為態度不好。
**STRIKE 改造重點**：R（擅長向上溝通的資深職場人）+ K（語氣：先認同→再說明→提替代方案。禁止說「我不想加班」）

---

## E4 — 催主管簽核拖了兩週的案子

**情境**：一份專案預算案卡在主管那兩週了，下游進度全被擋住。

```
主管好，

那個預算案可以請您看一下嗎？
已經送出一段時間了。

謝謝。
```

**為什麼「爛」**：沒有急迫感（「一段時間」多久？）、沒說影響範圍、語氣太弱。
**STRIKE 改造重點**：S（影響下游：哪些事被擋住、延遲多少天）+ K（不卑不亢、點出影響→請求時程→表達配合意願）

---

## E5 — 通知團隊專案時程變更

**情境**：客戶臨時追加需求，交付日從 3/20 延到 4/5，你要通知團隊。

```
各位好，

專案時間有一些調整，
新的時間之後再跟大家確認。

先讓大家知道一下。
```

**為什麼「爛」**：沒說改到什麼時候、沒說為什麼改、沒說對各人工作的影響、「之後再確認」= 等於沒確認。
**STRIKE 改造重點**：I（原時程、新時程、變更原因、各人受影響的工作項目）+ E（格式範例：表格列出各人負責項目的新截止日）
