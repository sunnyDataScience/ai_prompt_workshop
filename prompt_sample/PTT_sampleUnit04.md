## 📂 第一單元：核心 Methodology —— STRIKE 指令架構

在 2026 年的企業環境中，與 AI 溝通的品質決定了決策的品質。我們統一採用 **STRIKE** 架構作為所有指令（Prompt）的基準。

|                         |                | **範例應用**                                 |
| ----------------------- | -------------- | -------------------------------------------------- |
| **S (Situation)** | **情境** | 目前企業面臨的外部挑戰或內部痛點。                 |
| **T (Task)**      | **任務** | 指定 AI 產出具體成果（調查報告、PPT 大綱、潤色）。 |
| **R (Role)**      | **角色** | 定義 AI 的專家身份（數據科學家、COO、HR 顧問）。   |
| **I (Input)**     | **輸入** | 提供的核心數據、觀察指標或受眾背景。               |
| **K (KPI)**       | **標準** | 邏輯架構（如金字塔原理）、字數、語氣風格。         |
| **E (Example)**   | **範例** | 期望的輸出格式或參考標竿。                         |

### 🚀 議題三：【價值重構研究】「AI 代理人」導入後的資深員工「經驗權重」量化評估

### 📖 故事背景 (Background)

隨著 2026 年 AI Agent 能夠獨立執行 90% 的例行事務，30-50 歲的資深員工正面臨一場「被評價」的戰役。許多高層認為：既然 AI 都能做了，為什麼還要付高薪給資深員工？這份報告旨在反駁這種偏見，透過研究證明：在 2026 年，資深員工的價值在於**「AI 決策審計」 與 「風險權重分配」**。人類的經驗是 AI 盲點的最後防線，這份研究將量化這種「無形經驗」的價值。

### 🛠️ 四階段 STRIKE 指令流

#### **Phase 0：深度調查 (The Researcher)**

**(S) Situation:** 我正在研究【「AI 代理人」導入後的資深員工「經驗權重」量化評估】，探討導入 AI Agent 後，資深工程師的「經驗」如何被重新定價與量化。
**(T) Task:** 請調查全球前瞻科技企業，在導入程式生成或 Data Agent 後，資深與資淺員工的價值貢獻比例變化。
**(R) Role:** 你是一位專精於 AI 轉型與技術人力資本（Tech Human Capital）的研究員。
**(I) Input:** 重點關注：架構決策時間佔比提升、Prompt 工程對除錯效率的影響、資深員工產出槓桿率（Leverage Ratio）的變化。
**(K) KPI:** 提供 3 個具備統計意義的量化指標，總結 2 個核心洞察。
**(E) Example:** 格式請參考：【趨勢現象】→【數據支撐】→【對企業的風險/機會點】。

#### **Phase 1：架構設計 (The Architect)**

**(K) KPI:** 1. 符合金字塔原理；2. 標題為強烈論點；3. 語氣設定為前瞻性、具權威感。

#### **Phase 2：核對與潤色 (The Supervisor)**
