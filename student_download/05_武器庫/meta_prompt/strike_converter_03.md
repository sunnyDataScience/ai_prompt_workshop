# STRIKE 轉換器：把你的想法一次變成可重複用的結構化 Prompt

> **前情提要**：
> - `meta_prompt.md` 教你怎麼跟 AI 協作（想清楚 → 講出來 → 補一句）
> - `meta_prompt_consolidation.md` 教你怎麼從過去的對話紀錄收割成模板
>
> **這篇解決的問題不一樣**：你是專家，你腦中已經有很清楚的需求和流程。你不想來回 5 輪再收割——你想**一次到位**，直接把腦中的東西轉成可重複用的 STRIKE 模板。

---

## 為什麼不能直接說「幫我用 STRIKE 格式寫」？

你可能試過這樣：

```
幫我用 STRIKE 格式把以下需求整理成結構化 prompt：
[貼上你的需求]
```

AI 會回你一個看起來有 S-T-R-I-K-E 標籤的東西。但通常有三個問題：

| 問題 | 原因 |
|------|------|
| AI 不知道 STRIKE 是什麼 | STRIKE 不是業界通用術語，AI 可能瞎猜或跟別的框架搞混 |
| 產出是「一次性」的 | AI 會把你的具體內容直接寫死，而不是做成有 `[變數]` 的可重複模板 |
| 缺乏品質護欄 | AI 不會自動幫你加防呆句、驗收標準，因為它不知道你的行業地雷 |

**解法**：你不需要學怎麼寫完美的 STRIKE，你需要一段「轉換器 Prompt」——它負責教 AI 什麼是 STRIKE、怎麼分辨固定 vs 變動的部分、以及你的品質要求。

---

## 整件事就兩步

```
腦中的想法（打字或語音轉文字）──→ 貼進轉換器 ──→ 拿到可重複用的 STRIKE 模板
```

---

## 第一步：把你腦中的東西倒出來

不用管格式，不用管結構。用你最自然的方式把以下幾件事講出來：

- 你在做什麼事、為什麼要做
- 你希望 AI 幫你產出什麼
- 你的標準是什麼（什麼叫好、什麼不能出錯）
- 如果有的話，你過去做這件事的流程是什麼

**打字或語音都行**。語音的話，用手機錄完丟給任何語音轉文字工具（內建聽寫、Whisper、Google 語音輸入都行），不用修逐字稿的錯字，AI 看得懂。

### 倒出來的東西大概長這樣

```
我每個月要做一份部門月報給副總。
資料是從 ERP 拉的，有各產品線的銷售額、退貨、毛利。
我需要 AI 幫我整理成副總看得懂的報告。

副總關心的就三件事：哪個產品線賺錢、哪個在虧、跟上個月比有什麼變化。
他不看細節，只看結論和趨勢。

格式的話，我習慣先一段摘要，然後一張總表，最後三條行動建議。
摘要要結論先行，不要鋪陳。
表格要有跟上月比的箭頭或百分比。

重點是數字不能錯，毛利率的算法是（銷售額-成本）/銷售額，
不是 AI 自己亂算的。如果某個數據我沒給，不要幫我猜。

每個月的資料格式都一樣，就是產品線名稱、銷售額、成本、退貨數這幾個欄位。
```

你看，很散、很口語、有重複——**完全沒關係**。這就是你腦中的原始素材。

## 第二步：貼進轉換器

把第一步的內容，連同下面這段「轉換器 Prompt」一起貼給 AI。

### STRIKE 轉換器 Prompt（直接複製使用）

```
你是一個提示詞架構師。我會給你一段未整理的需求描述（可能是口語、
可能有重複、可能很散亂）。請你幫我轉換成一個「可重複使用的 STRIKE
結構化提示詞模板」。

=== STRIKE 格式定義 ===

STRIKE 有六個要素，請嚴格依照這個結構輸出：

【S — 情境】交代背景脈絡，讓 AI 不用猜
【T — 任務】明確說出要 AI 做什麼，用動詞開頭
【R — 角色】指定 AI 的專業身分
【I — 輸入】說明會提供什麼資料
【K — 驗收指標】定義什麼叫「好」，包含格式、品質、禁止事項
【E — 範例】給出期望的輸出樣本

=== 轉換規則 ===

1. 分辨「固定」vs「變動」：
   - 每次都一樣的內容 → 直接寫死在模板裡
   - 每次會變的內容 → 用 [方括號＋說明] 標記
   例如：「我是 [部門] 的 [職稱]」是變動的，
   「數字不能自行推估」是固定的

2. K（驗收指標）要特別完整：
   - 把使用者提到的所有品質標準、格式要求、禁止事項都收進 K
   - 如果使用者提到計算公式，寫進 K（例如「毛利率 =（銷售額-成本）/ 銷售額」）
   - 加一條通用防呆：「若資料不足以完成某項分析，標註 [資料缺失] 而非自行推估」

3. E（範例）要具體：
   - 根據使用者描述的格式偏好，產出一段「填好的範例」
   - 讓未來使用這個模板的人一眼知道輸出該長什麼樣

4. 模板最後附上：
   - 一段「使用說明」：列出每次使用前要準備什麼、要替換哪些 [方括號]
   - 一段「填好的範例」：把 [方括號] 實際填入一次，展示完成品

=== 我的需求描述 ===

"""
[把你第一步倒出來的內容貼在這裡]
"""
```

## 你會拿到什麼

AI 會幫你產出一份完整的 STRIKE 模板。以月報案例為例：

```
=== STRIKE 模板：部門月度報告 ===

【S — 情境】
我是 [部門名稱] 的 [職稱]，正在準備 [月份] 月的部門月報。
報告對象是副總，他關注的重點是：
哪個產品線賺錢、哪個在虧、跟上月相比的變化趨勢。

【T — 任務】
請根據以下資料，產出一份月度報告，依序包含：
1. 管理摘要（3-5 句，結論先行，不要鋪陳）
2. 產品線績效總表（含與上月比較）
3. 行動建議（3 條，具體可執行）

【R — 角色】
你是資深財務分析師，擅長把數據轉化為高管能快速決策的報告，
風格簡潔、結論導向。

【I — 輸入】
本月資料（CSV 格式）：
"""
[貼上本月 ERP 匯出資料，欄位：產品線、銷售額、成本、退貨數]
"""
上月資料：
"""
[貼上上月資料，格式同上]
"""

【K — 驗收指標】
- 管理摘要用「結論先行」格式：先講最重要的發現，再補數據佐證
- 總表包含：產品線、銷售額、成本、毛利率、MoM 變化率
- 毛利率 =（銷售額 - 成本）/ 銷售額，取至小數點第一位
- MoM 變化率標註分子分母
- 數字加千分位，金額單位為新台幣
- 表格控制在 5-6 欄以內
- 所有數字必須來自我提供的資料，不可自行推估
- 若資料缺失，標註 [資料缺失] 而非自行補數
- 行動建議必須基於數據發現，不可泛泛而談

【E — 範例】
管理摘要範例：
「3 月整體營收較上月成長 8.2%（$12,500K → $13,525K），主要由 A 產品線
帶動（+15.3%）。B 產品線連續兩月衰退（-4.1%），毛利率降至 18.2%，
建議檢視定價策略。C 產品線維持穩定。」

總表範例：
| 產品線 | 銷售額(千元) | 成本(千元) | 毛利率 | MoM 變化 |
|--------|------------|-----------|--------|---------|
| A 產品線 | 6,200 | 3,720 | 40.0% | +15.3% |

---

使用說明：
每次使用前，替換以下 [方括號] 內容：
1. [部門名稱] — 你的部門
2. [職稱] — 你的職稱
3. [月份] — 本次報告的月份
4. [貼上本月 ERP 匯出資料] — 從 ERP 匯出的 CSV
5. [貼上上月資料] — 上月同格式資料

K 和 E 區塊不需要改，除非副總的關注重點有變化。
```

## 進階用法：讓模板自己進化

模板用了幾次之後，你可能發現某些地方每次都要手動調。這時候你可以再跟 AI 說一句：

```
我這個模板已經用了三次，每次都要額外補充以下要求：
- [你每次補充的那句話]
- [另一句]

請幫我把這些直接寫進模板的 K（驗收指標）裡，這樣下次就不用再補。
```

**模板不是寫完就不動的**。用幾次、補幾句、越用越準。這跟你第一次寫 SOP 然後用著用著會改一樣。

## 語音輸入的實戰建議

如果你習慣用講的比用打的快，整個流程會變成：

```
對著手機講 2 分鐘 ──→ 語音轉文字 ──→ 不用修，直接貼進轉換器 ──→ 拿到模板
```

幾個小提醒：

| 你擔心的 | 實際上 |
|---------|--------|
| 逐字稿很多錯字 | AI 完全看得懂，「副總」變成「副中」它也猜得到 |
| 講得很散、有重複 | 轉換器的工作就是幫你整理，散和重複正常 |
| 講到一半改口 | 沒關係，AI 會取你最後確認的版本 |
| 我不知道 STRIKE 六個字母 | 你不需要知道，轉換器會幫你分類 |

**你唯一要做的**：把你腦中「這件事怎麼做」講出來。結構化是 AI 的工作，不是你的。

## 三份文件的關係

```
你跟 AI 的互動能力

Level 1 — 日常使用（meta_prompt.md）
  想清楚 → 講出來 → 不對就補一句
  適合：所有人、所有場景

Level 2 — 事後收割（meta_prompt_consolidation.md）
  回頭看 → 挑重點 → 請 AI 整合 → 測一次
  適合：已經來回多輪、以後會重複做的任務

Level 3 — 一次到位（strike_converter.md ← 你在這裡）
  腦中想法倒出來 → 貼進轉換器 → 直接拿到 STRIKE 模板
  適合：領域專家、對需求很清楚、不想來回的人
```

不是說 Level 3 比較厲害。是不同情境用不同的：

- 隨便問個事 → Level 1 就好
- 做完才發現以後會再用 → Level 2 收割
- 你很清楚要什麼、想一步到位 → Level 3

## 一句話總結

> **你不需要學會寫 STRIKE。你只需要把你知道的事情講出來，讓轉換器幫你結構化。**
> **專家的價值在腦中的判斷和標準，不在提示詞的格式。**
